# Vive-Writing プロジェクト構造設計

## プロジェクト概要

Vive-Writingは、Claude Codeを使用して音声指示のみで高品質なウェブ記事を生成する「バイブライティング」システムです。ユーザーは音声で指示を出すだけで、参考記事のトーンや文体を学習したAIが質の高い記事を自動生成します。

## ディレクトリ構造

```
Vive-Writing/
├── input/                    # 音声入力・指示ファイル
│   ├── audio/               # 音声ファイル保存場所
│   ├── transcripts/         # 文字起こしテキスト
│   └── instructions/        # 記事作成の指示・要件
├── output/                   # 生成された記事
│   ├── drafts/              # 下書き
│   ├── published/           # 完成記事
│   └── archive/             # アーカイブ
├── source/                   # 参考記事・スタイルガイド
│   ├── reference-articles/  # 参考記事コレクション
│   ├── tone-guides/         # トーン・文体ガイド
│   └── style-samples/       # スタイルサンプル
├── config/                   # 設定ファイル
│   ├── prompts/             # プロンプトテンプレート
│   ├── settings/            # 各種設定
│   └── workflows/           # ワークフロー定義
├── scripts/                  # 自動化スクリプト
│   ├── preprocessing/       # 前処理スクリプト
│   ├── generation/          # 記事生成スクリプト
│   └── postprocessing/      # 後処理スクリプト
└── docs/                     # ドキュメント
    ├── guides/              # 使い方ガイド
    ├── examples/            # サンプル記事
    └── api/                 # API仕様書
```

## 各ディレクトリの詳細説明

### 1. `input/` - インプットフォルダ

記事作成のための入力データを格納します。

- **`audio/`**: 音声ファイル（.mp3, .wav, .m4a等）を保存
- **`transcripts/`**: 音声から文字起こしされたテキストファイル
- **`instructions/`**: 記事のテーマ、キーワード、構成などの指示書（Markdown形式推奨）

**ファイル命名規則例**:
- `input/audio/2026-01-21_article-topic.mp3`
- `input/transcripts/2026-01-21_article-topic.txt`
- `input/instructions/2026-01-21_article-topic.md`

### 2. `output/` - アウトプットフォルダ

生成された記事を管理します。

- **`drafts/`**: AIが生成した初期の下書き
- **`published/`**: レビュー・編集を経た完成記事
- **`archive/`**: 古い記事や不要になった下書きのアーカイブ

**ワークフロー**: drafts → (レビュー・編集) → published

### 3. `source/` - ソースフォルダ

記事の品質とトーンを決定する参考資料を格納します。

- **`reference-articles/`**:
  - 優れた記事の実例
  - カテゴリー別に整理（例: tech/, lifestyle/, business/）
  - AIがこれらを学習してトーンや構成を理解

- **`tone-guides/`**:
  - トーン・ボイスのガイドライン
  - フォーマル/カジュアル、専門的/一般向けなどの指針

- **`style-samples/`**:
  - 特定のライティングスタイルのサンプル
  - 導入部、本文、結論の書き方例

**重要**: このフォルダの内容がAIの出力品質に直接影響します。高品質な参考記事を充実させることが成功の鍵です。

### 4. `config/` - 設定フォルダ

システムの動作を制御する設定ファイル群。

- **`prompts/`**:
  - Claude用のプロンプトテンプレート
  - 記事タイプ別のプロンプト（レビュー、ハウツー、ニュース等）

- **`settings/`**:
  - 文章の長さ、トーン、ターゲット読者などの設定
  - JSON/YAMLフォーマット

- **`workflows/`**:
  - 記事生成ワークフローの定義
  - ステップバイステップの処理フロー

### 5. `scripts/` - スクリプトフォルダ

自動化とワークフローを実現するスクリプト。

- **`preprocessing/`**:
  - 音声ファイルの文字起こし
  - テキストのクリーニング・整形

- **`generation/`**:
  - Claude APIとの連携
  - 記事生成の実行スクリプト

- **`postprocessing/`**:
  - 生成記事の整形
  - メタデータの追加
  - 品質チェック

### 6. `docs/` - ドキュメントフォルダ

プロジェクトのドキュメント類。

- **`guides/`**: 使い方、セットアップガイド
- **`examples/`**: サンプル記事と生成プロセスの例
- **`api/`**: 使用するAPI（Claude API等）の仕様書

## 推奨ワークフロー

1. **準備**: `source/`に参考記事とスタイルガイドを配置
2. **インプット**: 音声で指示を録音 → `input/audio/`に保存
3. **文字起こし**: スクリプトで音声をテキスト化 → `input/transcripts/`
4. **指示書作成**: 文字起こしから指示書を生成 → `input/instructions/`
5. **記事生成**: Claudeが参考記事を基に記事を生成 → `output/drafts/`
6. **レビュー・編集**: 必要に応じて手動編集
7. **公開**: 完成記事を`output/published/`へ移動

## 技術スタック推奨

- **音声認識**: OpenAI Whisper API / Google Speech-to-Text
- **記事生成**: Claude API (Claude 3.5 Sonnet以上推奨)
- **スクリプト言語**: Python 3.10+
- **設定フォーマット**: YAML
- **記事フォーマット**: Markdown

## 拡張可能性

この構造は以下の拡張に対応できます：

- **多言語対応**: 各フォルダに言語別サブフォルダを追加
- **バージョン管理**: Gitで記事の変更履歴を追跡
- **SEO最適化**: メタデータ管理とキーワード分析機能
- **自動公開**: CMS連携による自動アップロード
- **品質評価**: 生成記事の自動評価システム

## ベストプラクティス

1. **参考記事の品質**: `source/`の内容を定期的に更新・改善
2. **命名規則の統一**: 日付ベースの命名で管理を容易に
3. **バージョン管理**: Gitで全ての変更を追跡
4. **テンプレート活用**: `config/prompts/`のテンプレートを充実させる
5. **定期的なレビュー**: 生成された記事の品質を継続的に評価

## まとめ

この構造により、音声入力から高品質記事の生成までのプロセスを効率化し、スケーラブルで保守しやすいシステムを実現できます。
