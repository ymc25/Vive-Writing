# Vive-Writing 入門ガイド

## はじめに

Vive-Writingへようこそ！このガイドでは、音声指示から高品質な記事を生成するまでの手順を、ステップバイステップで説明します。

## 前提条件

- 音声録音ができるデバイス（スマートフォン、PCなど）
- Claude API キー（取得方法は[こちら](https://www.anthropic.com/)）
- 基本的なファイル操作の知識

## ステップ1: 参考記事を準備する

記事の品質は、参考記事の質に大きく依存します。

### 推奨する参考記事の選び方

1. **高品質な記事を選ぶ**
   - 文章が明確で読みやすい
   - 構成が論理的
   - 情報が正確で信頼できる

2. **自分のスタイルに近い記事を選ぶ**
   - 目指すトーンやボイスに近い
   - ターゲット読者層が似ている

3. **複数の記事を用意する**
   - 最低3〜5記事を推奨
   - 多様性を持たせることで、より柔軟な生成が可能

### 参考記事の配置方法

```bash
# 参考記事をsource/reference-articles/に配置
# カテゴリー別にサブフォルダを作ることを推奨

source/reference-articles/
├── tech/
│   ├── ai-trends.md
│   └── productivity-tools.md
├── business/
│   ├── remote-work.md
│   └── management-tips.md
└── lifestyle/
    ├── work-life-balance.md
    └── health-tips.md
```

## ステップ2: トーンガイドを確認・調整する

`source/tone-guides/`にあるトーンガイドを確認し、必要に応じて調整します。

デフォルトでは以下のトーンガイドが用意されています：
- `professional-friendly.md`: プロフェッショナルかつ親しみやすい（推奨）

## ステップ3: 音声で指示を録音する

### 効果的な音声指示のポイント

1. **明確なテーマを伝える**
   ```
   「今日は〇〇についての記事を書きたいです」
   ```

2. **キーワードを指定する**
   ```
   「キーワードは△△、□□、そして××です」
   ```

3. **文字数の目安を伝える**
   ```
   「2000文字程度で書いてください」
   ```

4. **ターゲット読者を明示する**
   ```
   「対象読者はビジネスパーソンです」
   ```

5. **希望する構成を伝える**
   ```
   「導入、問題提起、解決策、まとめという構成でお願いします」
   ```

### 音声指示の例

```
「今日は2026年のAIトレンドについての記事を書きたいです。
特に生成AIの進化と、それがビジネスに与える影響について
3000文字程度で、初心者にも分かりやすく説明してください。
キーワードは、生成AI、ビジネス活用、自動化です。
対象読者はビジネスパーソンと技術者です。
構成は、導入、現状分析、トレンド紹介、ビジネスへの影響、
まとめという流れでお願いします。」
```

### 録音ファイルの保存

```bash
# 録音ファイルをinput/audio/に保存
# 命名規則: YYYY-MM-DD_topic-name.mp3

input/audio/2026-01-21_ai-trends.mp3
```

## ステップ4: 文字起こし（現在準備中）

※ 現在、文字起こしスクリプトは開発中です。
一時的に、以下の方法で文字起こしを行ってください：

1. **手動で文字起こし**
   - 音声を聞きながらテキスト化
   - `input/transcripts/`に保存

2. **外部サービスを利用**
   - Google Docs音声入力
   - Whisper API
   - その他の文字起こしサービス

## ステップ5: 指示書を作成

文字起こしテキストを基に、構造化された指示書を作成します。

### 指示書のテンプレート

```markdown
# 記事作成指示書

## 基本情報
- 作成日: 2026-01-21
- テーマ: 2026年のAIトレンド

## 記事詳細
- タイトル案: （必要に応じて）
- キーワード: 生成AI、ビジネス活用、自動化
- 文字数: 3000文字程度
- トーン: professional-friendly

## ターゲット読者
- ビジネスパーソン
- 技術者（初心者〜中級者）

## 構成
1. 導入: AIの現状と注目の背景
2. 現状分析: 2026年初頭のAI状況
3. トレンド紹介: 主要な3〜5つのトレンド
4. ビジネスへの影響: 具体的な活用例
5. まとめ: 今後の展望と行動提案

## 追加要件
- 具体的な数字やデータを含める
- 実例を2〜3件紹介する
- 初心者にも理解できる説明を心がける
```

保存先: `input/instructions/2026-01-21_ai-trends.md`

## ステップ6: 記事を生成（現在準備中）

※ 現在、自動生成スクリプトは開発中です。
一時的に、Claude Codeを直接使用して記事を生成してください：

1. 参考記事を読み込む
2. トーンガイドを確認する
3. 指示書の内容をClaudeに伝える
4. 生成された記事を`output/drafts/`に保存

## ステップ7: レビュー・編集

生成された記事を確認し、必要に応じて編集します。

### チェックポイント

- [ ] 事実確認: 情報が正確か
- [ ] 文体確認: 一貫したトーンか
- [ ] 構成確認: 論理的な流れか
- [ ] 読みやすさ: 理解しやすい表現か
- [ ] SEO確認: キーワードが適切に含まれているか

### 編集のヒント

- 不自然な表現を修正
- 具体例を追加
- データや数字の正確性を確認
- 見出しを最適化

## ステップ8: 公開準備

編集が完了したら、記事を公開準備フォルダに移動します。

```bash
# 完成した記事をpublishedフォルダに移動
mv output/drafts/2026-01-21_ai-trends.md output/published/
```

## トラブルシューティング

### Q: 生成された記事のトーンが希望と異なる

A: 以下を確認してください：
- 参考記事が希望のトーンに合っているか
- トーンガイドが明確に定義されているか
- 指示書でトーンを明示しているか

### Q: 記事の品質が低い

A: 以下を試してください：
- より高品質な参考記事を追加する
- 指示をより具体的にする
- 複数回生成して比較する

### Q: 文字数が希望と大きく異なる

A: 指示書で明確に文字数を指定し、「厳守してください」と追記します。
