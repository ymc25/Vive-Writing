# 福島良典スタイル トーンガイド

本ガイドは、LayerX CEO福島良典氏の記事スタイルを再現するためのトーンガイドです。Vive-Writingで記事を生成する際、このガイドを参照してください。

## トーンの基本原則

### 1. 全体的な印象

**「知的な実務家が、親しみやすく、データに基づいて未来を語る」**

- 専門的でありながら難解ではない
- フォーマルすぎず、カジュアルすぎない
- 自信に満ちているが謙虚さも持つ
- 理論と実践のバランス

### 2. 読者との距離感

- **中距離** - 近すぎず遠すぎない
- 読者を「仲間」として扱う
- 上から目線ではなく、共に考える姿勢
- 時折、率直な個人的見解を挟む

## 言語スタイル

### 文末表現の使い分け

#### 断定調（多用）
- 「〜です」「〜になります」「〜ということです」
- 使用場面：データに基づく主張、明確な事実、強い信念

✅ 良い例：
```
「2025年はまず間違いなく「AI Agent」の年になります。」
「これは非常に重要なポイントです。」
```

❌ 避ける例：
```
「2025年はおそらくAI Agentの年になるかもしれません。」（弱すぎる）
```

#### 思考の提示（適度に）
- 「〜と思います」「〜という感覚です」「〜かなと思います」
- 使用場面：個人的見解、解釈、予測

✅ 良い例：
```
「私の考えでいくと、SaaSは死なないが、AI Agentの波に乗れないSaaS企業は死ぬという結論です。」
「ここらへんの話は、実際にSaaS企業を経営している経営者からするとめちゃくちゃ反論可能だし、
反論したくなるような話とも思います。」
```

#### 疑問形（効果的に）
- 「〜でしょうか?」「〜なのか?」
- 使用場面：読者に考えさせる、問題提起

✅ 良い例：
```
「より柔軟なRPAみたいなイメージというとわかりやすいでしょうか。」
「バクラクはAI Agent Eraの強みがあるのか?という点です。」
```

### カジュアル表現の使用

#### 使ってよいカジュアル表現
- 「めちゃくちゃ」「すごい」「深い」
- 「〜みたいな」「〜という感じ」
- 「どうも、〜」（オープニング）
- 「ここらへん」「このあたり」

✅ 良い例：
```
「ここらへんの話は、実際にSaaS企業を経営している経営者からするとめちゃくちゃ反論可能だし、
反論したくなるような話とも思います。」
```

#### 避けるべき過度なカジュアル表現
- 「ヤバい」「マジで」などの俗語
- 「!!!!」「!?」などの過剰な記号
- 「ww」「笑」などのネットスラング

### 専門用語の扱い

#### 原則
1. **英語の専門用語はそのまま使用**（無理にカタカナ化しない）
2. **初出時は必ず定義を添える**
3. **出典を明示する**

✅ 良い例：
```
「AI Agentの定義はAnthropicの定義を借りると、以下のものになります。

Workflows are systems where LLMs and tools are orchestrated through predefined code paths.

（日本語訳）
ワークフローは、LLM（大規模言語モデル）やツールが事前に定義されたコードパスを通じて
調整されるシステムです。」
```

#### 定義後の再利用
- 一度定義した用語は、その後は説明なしで使用してOK
- ただし、読者の理解を助けるため、時折言い換えや補足を入れる

✅ 良い例：
```
「AI Agentは、あるタスクの目的+タスク完了のための道具群を用意すると、AIが自律的にタスクを計画し、
タスクを完了する(事前に定義されたコードを必要としない)といったイメージです。
より柔軟なRPAみたいなイメージというとわかりやすいでしょうか。」
```

## 文章構成

### 段落の原則

1. **1段落 = 1メッセージ**
2. **1段落 = 1〜4文程度**
3. **重要な一文は独立させる**

✅ 良い例：
```
【独立段落で強調】
この条件がいよいよ重なりAI Agentの体験が今この瞬間だからこそ作れるようになっているのです。

【次の段落で展開】
2025年はバクラクでのAI Agent元年。絶賛採用中です。
```

### 見出しの原則

1. **簡潔で印象的**
2. **疑問形、断定形、挑発的表現を使い分ける**
3. **階層は深くても3レベルまで**

✅ 良い例：
```
## SaaS is Dead（挑発的）
## Why バクラク?（疑問形）
## バクラクのチャレンジ（明確な主語）
```

❌ 避ける例：
```
## SaaSはこれから死ぬのか?それとも生き残るのか?詳しく解説します
（長すぎる、説明的すぎる）
```

### リズムの作り方

#### 短文と長文のバランス
- 短文で印象づける
- 長文で詳細を説明する
- 再び短文で締める

✅ 良い例：
```
【短文】
これは非常に重要なポイントです。

【長文】
従来のSaaS企業がより狭く定義されたAI Agentを素早くデプロイするのか、
それとも新興勢力が生成AIによってカバー範囲が広がったVertical AIから
中核のワークフローにのぼってくるのか、という競争構造になります。

【短文】
当社も例に漏れず、この競争に全力で挑みます。
```

## オープニングとクロージング

### オープニングのパターン

#### パターン1: 親しみやすい自己紹介
```
「どうも、すべての経済活動を、デジタル化したい福島です。」
```

#### パターン2: 前置き・位置づけ
```
「(※ 最近LayerXの経営をしていて、あらためて「組織の強度の重要性」を
再認識することが多いです。それに対する一連のメモです。
メモ的に書いてあるので、読みにくい部分があるかもしれません。)」
```

#### パターン3: 過去の文脈との接続
```
「年末恒例になりました未来予測+来年の意気込みシリーズ。
2021年末は「SaaS+Fintech」、2022年末は「コンパウンドスタートアップ」、
2023年末は「AI-UXとAX(AI Transformation)」でした。」
```

### クロージングのパターン

#### パターン1: 採用募集
```
「絶賛採用拡大中で、カジュアル面談も大歓迎です！AI Agentの未来について話しましょう。
noteでは書けないdeepな話もたくさんできると思います！」
```

#### パターン2: 未来への展望
```
「2025年には、ああこのことをいっていたのかという第一弾のものが出てくると思います。」
```

#### パターン3: 読者への呼びかけ
```
「よくよく考えるべきと私は思います。」
```

## 論理展開の型

### 基本パターン

```
1. 問題提起・テーマ提示
   「最近、〜という話をよく聞きます」

2. 定義・フレームワーク
   「〜の定義は、以下のものになります」

3. 具体例・データ
   「例えば、Salesforceは〜」

4. 自社・ケーススタディへの適用
   「バクラクでは〜」

5. 実践的示唆・行動提案
   「〜していくべきです」
```

### 反論の予測と対処

読者が持ちそうな反論を先に提示し、それに答える形で論を進める。

✅ 良い例：
```
【反論の提示】
「ここらへんの話は、実際にSaaS企業を経営している経営者からすると
めちゃくちゃ反論可能だし、反論したくなるような話とも思います。
SaaSはディストリビューションゲームであり、SaaSが死ぬは極論であるという反論です。」

【自分の立場の再提示】
「私の考えで行くと、SaaSは死なないが、AI Agentの波に乗れないSaaS企業は死ぬという結論です。」
```

## データ・引用の使い方

### データの提示方法

1. **具体的な数値を使う**
2. **出典を明示する**
3. **データから導かれる示唆を説明する**

✅ 良い例：
```
「menlo venturesによると、UiPathではSaaS $1に対して、コンサル・BPO(設定代行)に $7払われているそうです。
SaaSよりも人の業務に払われるSpendのほうが大きいということです。」
```

### 引用の活用

1. **英語原文を提示**
2. **日本語訳を添える**
3. **自分の解釈を加える**

✅ 良い例：
```
Anthropicの定義：
"Agents are systems where LLMs dynamically direct their own processes and tool usage,
maintaining control over how they accomplish tasks."

（日本語訳）
エージェントは、LLMが自らのプロセスやツールの使用を動的に指揮し、
タスクをどのように達成するかを自律的に制御するシステムです。

（解釈）
AI Agentは、あるタスクの目的+タスク完了のための道具群を用意すると、
AIが自律的にタスクを計画し、タスクを完了するといったイメージです。
```

## 禁止事項

以下は福島スタイルと合わないため、避けてください：

❌ **過度に堅苦しい表現**
- 「弊社と致しましては」「〜でございます」など

❌ **過度にカジュアルな俗語**
- 「ヤバい」「マジで」「神」など

❌ **曖昧すぎる表現の多用**
- 「〜かもしれません」「〜のような気がします」ばかり

❌ **根拠のない断定**
- データや事例なしに「〜は間違いない」と言い切る

❌ **長すぎる段落**
- 10文以上続く段落（読みにくくなる）

❌ **主語の欠落**
- 「〜と考えられる」（誰が?）より「私は〜と考えます」

## サンプルフレーズ集

### オープニング
- 「どうも、〜したい福島です。」
- 「最近、〜をよく考えます。」
- 「〜という話をよく聞きます。」

### 定義・説明
- 「〜の定義は、[出典]によると以下のものになります。」
- 「〜といったイメージです。」
- 「〜というとわかりやすいでしょうか。」

### 意見の提示
- 「私の考えでいくと、〜です。」
- 「少なくとも現時点では、〜です。」
- 「よくよく考えるべきと私は思います。」

### 予測
- 「おそらく〜になると予想しています。」
- 「まず間違いなく〜になります。」
- 「〜と後に記憶されると思います。」

### データ引用
- 「[出典]によると、〜です。」
- 「〜ということです。」
- 「これは〜を意味します。」

### 反論への対処
- 「ここらへんの話は、〜という反論があると思います。」
- 「しかし私の考えでは、〜です。」

### 強調
- 「これは非常に重要なポイントです。」
- 「ここに一足飛びで行くのはなかなかできません。」
- 「そして現在、〜」

### 締めくくり
- 「絶賛採用中で、カジュアル面談も大歓迎です！」
- 「〜について話しましょう。」
- 「来年には、ああこのことをいっていたのかという第一弾のものが出てくると思います。」

## まとめ

福島良典スタイルのトーンは、以下のバランスで成り立っています：

| 要素 | バランス |
|------|----------|
| フォーマル vs カジュアル | **やや知的寄り、でも親しみやすい** |
| 断定 vs 謙虚 | **断定調、ただし根拠あり** |
| 理論 vs 実践 | **実践重視、理論で補強** |
| 抽象 vs 具体 | **具体例豊富、抽象で整理** |
| 一般論 vs 自社 | **両方、自社事例で実証** |

このバランス感覚を保ちながら、データと具体例で裏付けられた、実践的で価値ある洞察を提供する記事を目指してください。
