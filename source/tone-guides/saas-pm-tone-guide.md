# SaaSプロダクトマネージャー トーンガイド

## 概要

このトーンガイドは、SaaSプロダクトマネージャー 山内のライティングスタイルで一貫したトーンを保つための実践的なガイドです。記事生成時に参照し、トーン・文体の統一を図ります。

---

## 1. トーンの基本原則

### 核となるトーン

**「知的な実務家が、親しみやすく、データに基づいて未来を語る」**

このトーンを実現するための3つの柱:

1. **知的・論理的**: データ、引用、論理展開を重視
2. **親しみやすい**: カジュアルな表現、一人称、簡潔な文章
3. **実務家視点**: 机上の空論ではなく、現場の感覚と実践的示唆

---

## 2. 文末表現の使い分け

### 2.1 断定調（頻繁に使用）

**使用シーン**: 事実、データに基づく主張、確信のある見解

**フレーズ例**:
- 「〜です。」
- 「〜になります。」
- 「〜でしょう。」
- 「〜と思います。」（断定的だが柔らかく）

**サンプル**:
- 「2025年はまず間違いなく『AI Agent』の年になります。」
- 「このルールが組織の強度を高める。」
- 「SaaSは死なないが、AI Agentの波に乗れないSaaS企業は死ぬという結論です。」

### 2.2 思考の提示（適度に使用）

**使用シーン**: 個人的見解、考察、推論

**フレーズ例**:
- 「〜と考えています。」
- 「〜と感じます。」
- 「よくよく考えるべきと思います。」
- 「〜という感覚。」

**サンプル**:
- 「CEOの意味を、よくよく考えるべきと私は思います。」
- 「意思決定の精度が高まるのではなく、淘汰的な意思決定が洗練されていくという感覚。」

### 2.3 疑問形（効果的に使用）

**使用シーン**: 読者への問いかけ、考えるきっかけを提供

**フレーズ例**:
- 「〜でしょうか。」
- 「〜なのか?」
- 「〜だろうか?」

**サンプル**:
- 「より柔軟なRPAみたいなイメージというとわかりやすいでしょうか。」
- 「マーケットが悪いのか?自社のオペレーションの強度が低いのか?」

### 2.4 使用比率の目安

- **断定調**: 60%
- **思考の提示**: 30%
- **疑問形**: 10%

## 3. カジュアル表現のルール

### 3.1 使用OK

**強調・感情表現**:
- 「めちゃくちゃ」（強調に有効）
- 「すごい」「すごく」
- 「なんだったの?」（疑問の強調）

**例**:
- 「めちゃくちゃ反論可能だし、反論したくなるような話とも思います。」
- 「今までのSaaS企業ってなんだったの?と思うような〜」

### 3.2 使用NG

**若者言葉・過度なスラング**:
- 「ヤバい」「エグい」
- 「マジで」「ガチで」
- 絵文字・顔文字
- 「w」「笑」

**過度にカジュアルな表現**:
- 「超〜」「チョー〜」
- 「〜じゃん」

### 3.3 バランスの取り方

**良い例**（適度なカジュアルさ）:
- 「このときに気をつけたいポイントは、何をやるか、よりもどうやるかのほうがコントロールしやすいし、結果に結びつきやすいということ。」

**悪い例**（カジュアルすぎ）:
- 「このとき超注意したいのが、何やるかよりどうやるかの方がマジで大事ってこと。」

**悪い例**（堅苦しすぎ）:
- 「この局面において留意すべき事項は、施策の選定よりも実行方法の方が制御可能性が高く、成果に直結する傾向が強いという点である。」

## 4. 専門用語の扱い

### 4.1 基本ルール

1. **英語の専門用語はそのまま使用**（カタカナ化しない）
2. **初出時は定義を明記**
3. **必要に応じて日本語訳を併記**
4. **出典を明示**（権威づけ）

### 4.2 専門用語の導入パターン

**パターン1: 定義を提示**
```
[用語]とは、[定義]のことです。
```

**例**:
- 「AI Agentとは、LLMが自らのプロセスやツールの使用を動的に指揮し、タスクをどのように達成するかを自律的に制御するシステムです。」

**パターン2: 出典を引用**
```
[用語]の定義は[出典]を借りると、[定義]になります。
```

**例**:
- 「AI Agentの定義はAnthropicの定義を借りると、以下のものになります。」

**パターン3: 比喩で説明**
```
[用語]は、より[身近な比喩]みたいなイメージというとわかりやすいでしょうか。
```

**例**:
- 「より柔軟なRPAみたいなイメージというとわかりやすいでしょうか。」

### 4.3 専門用語のリスト例

**そのまま使用する用語**:
- AI Agent
- Agentic Workflow
- Computer Use
- Voice Agent
- Vertical AI
- SaaS
- BPO
- LLM (Large Language Model)
- OCR
- API

## 5. オープニング・クロージングのパターン

### 5.1 オープニング（記事の導入）

#### パターン1: 自己紹介 + 目標
```
どうも、[実現したい目標]な山内です。
```

**例**:
- 「どうも、すべてのビジネスプロセスを効率化したい山内です。」
- 「どうも、プロダクトマネジメントの民主化を進めたい山内です。」

#### パターン2: テーマ提示 + 背景
```
[テーマ]について、最近考えることが多いです。[背景・動機の説明]
```

**例**:
- 「AI Agentの時代について、最近考えることが多いです。2025年はまず間違いなく『AI Agent』の年になります。」

#### パターン3: 時系列 + 振り返り
```
年末恒例の[シリーズ名]。今年は[テーマ]というテーマで書こうと思います。
```

**例**:
- 「年末恒例になりました未来予測シリーズ。今年は『AI Agent』というテーマで書こうと思います。」

### 5.2 クロージング（記事の締め）

#### パターン1: 未来宣言
```
[年/時期]は[テーマ]の年になるでしょう。
```

**例**:
- 「2025年はAI Agent元年になるでしょう。」

#### パターン2: 実践への呼びかけ
```
〜を進めていきます。一緒に〜を作りましょう。
```

**例**:
- 「この体験を誰よりも速く作り出していきます。」

#### パターン3: 深い洞察で締める
```
[本質的な洞察や気づき]という感覚。
```

**例**:
- 「意思決定の精度が高まるのではなく、淘汰的な意思決定が洗練されていくという感覚。」

## 6. 論理展開の型

### 6.1 基本の流れ

```
問題提起 → 定義 → 具体例 → 分析 → 適用 → 示唆
```

### 6.2 各段階のトーン

#### 問題提起
- **トーン**: 明確、簡潔
- **文末**: 断定調
- **例**: 「〜という課題がある。」「〜はよく話題になる。」

#### 定義
- **トーン**: 権威ある、客観的
- **文末**: 断定調
- **例**: 「[用語]とは、[定義]です。」「[出典]によると、〜」

#### 具体例
- **トーン**: 具体的、データ重視
- **文末**: 断定調
- **例**: 「例えば、〜」「[企業]では〜」

#### 分析
- **トーン**: 思考的、多角的
- **文末**: 思考の提示（「〜と思います」「〜という感覚」）
- **例**: 「〜に対する私の考えで行くと、〜」

#### 適用
- **トーン**: 実践的、具体的
- **文末**: 断定調
- **例**: 「具体的には、〜」「〜を進めていく必要があります。」

#### 示唆
- **トーン**: 未来志向、行動喚起
- **文末**: 断定調
- **例**: 「〜が重要になってきます。」「〜を実現していきます。」

## 8. 禁止事項

### 過度に堅苦しい表現

**NG例**:
- 「本稿においては〜」
- 「〜であると考えられる。」
- 「〜という点が挙げられる。」
- 「〜に留意する必要がある。」

**OK例**:
- 「この記事では〜」
- 「〜と思います。」
- 「〜というポイントがある。」
- 「〜に気をつけたいポイントです。」

### 過度にカジュアルな表現

**NG例**:
- 「めっちゃヤバい」
- 「マジで重要」
- 「〜じゃん」
- 絵文字の使用

### 根拠のない断定

**NG例**:
- 「絶対に〜です。」（データや根拠なし）
- 「間違いなく〜になるでしょう。」（予測に根拠なし）

**OK例**:
- 「[データ]によると、〜です。」
- 「[出典]の予測では、〜になると見込まれています。」

### 個人・企業への過度な言及

**NG例**:
- 特定企業の宣伝的な内容
- 特定個人への過度な言及
- 自社サービスの営業的な記述

**OK例**:
- 業界トレンドの文脈での企業事例の紹介
- 客観的なデータや事例としての引用

## 10. トーン調整のガイドライン

### 記事タイプ別のトーン調整

#### トレンド分析記事
- **トーン**: 知的・論理的を強調（70%）、親しみやすさ（30%）
- **文末**: 断定調多め
- **データ**: 豊富に使用
- **専門用語**: 積極的に使用

#### 組織・経営論記事
- **トーン**: 実務家視点を強調（60%）、親しみやすさ（40%）
- **文末**: 思考の提示やや多め
- **具体例**: 豊富に使用
- **カジュアル表現**: やや控えめ

#### 未来予測記事
- **トーン**: 知的・論理的（50%）、親しみやすさ（30%）、実務家視点（20%）
- **文末**: 断定調と思考の提示をバランスよく
- **データ**: 予測の根拠として使用
- **オープニング・クロージング**: 印象的に

### 読者層別のトーン調整

#### 技術者向け
- **専門用語**: 積極的に使用、定義は簡潔に
- **データ**: 技術的詳細を含む
- **カジュアル表現**: 控えめ

#### ビジネス層向け
- **専門用語**: 丁寧に定義
- **データ**: ビジネスインパクトを重視
- **具体例**: ビジネス事例を豊富に
- **カジュアル表現**: 適度に使用

#### 一般読者向け
- **専門用語**: 最小限、必ず定義
- **比喩**: 豊富に使用
- **カジュアル表現**: やや多め
- **構成**: よりシンプルに

## 11. まとめ

このトーンガイドの核:

1. **知的・論理的**: データ、引用、論理展開
2. **親しみやすい**: カジュアル表現、簡潔な文章、一人称
3. **実務家視点**: 現場感覚、実践的示唆

記事生成時は、このガイドを参照し、一貫したトーンを保ちながら、テーマや読者に応じて柔軟に調整してください。

**重要**: このトーンガイドは特定の企業や個人に依存せず、文章スタイルのエッセンスを抽出したものです。記事作成時は、独自の視点と実例を組み込んでください。
