# ライティングスタイル・リファレンスガイド

## 概要

このドキュメントは、SaaSプロダクトマネージャー（山内）のライティングスタイルを体系化したリファレンスガイドです。分析対象の記事から抽出した文体・トーン・構成パターンを、今後の記事生成に活用できる形式で整理しています。

**重要**: このガイドは文体・トーン・構成パターンのみを参照します。特定の個人・企業への言及は含まれません。

---

## 1. 記事構成

### 1.1 オープニング（導入部）

**特徴:**
- 親しみやすく、個人的なトーンで始める
- 読者を引き込む問題提起や時代認識から入る
- カジュアルながら知的な印象を与える

**典型的なパターン:**

```markdown
どうも、[実現したいビジョン]したい[名前]です。

[時事的な背景・トレンド認識]
```

**実例に基づくバリエーション:**

1. **ビジョン提示型**
   - 「どうも、すべての経済活動を、デジタル化したい山内です。」
   - 「どうも、プロダクトの価値を最大化したい山内です。」
   - 「どうも、ビジネス課題をテクノロジーで解決したい山内です。」

2. **問題提起型**
   - 「最近、[テーマ]について、あらためて[気づき]を再認識することが多いです。」
   - 「今年は[トレンド]という大きな変化が起きています。」

3. **コンテキスト設定型**
   - 「年末恒例になりました未来予測シリーズ。今年は[テーマ]について書こうと思います。」
   - 「※ メモ的に書いてあるので、読みにくい部分があるかもしれません。」

### 1.2 論理展開パターン

**基本構造:**

```
1. 問題提起・時代認識
2. 定義・コンセプトの明確化
3. データ・具体例による裏付け
4. 自社/自分への適用
5. Why? の問いかけ（強みの説明）
6. 未来への展望・示唆
7. クロージング（CTA）
```

**各セクションの詳細:**

#### **1. 問題提起・時代認識**
- 現在起きている変化や課題を提示
- 「〜という言葉が盛り上がった」「〜が変わりつつある」
- タイムリーで読者が関心を持つトピック

#### **2. 定義・コンセプトの明確化**
- 重要な概念を明確に定義
- 権威ある出典（論文、有名企業、専門家）を引用
- 英語の専門用語はそのまま使い、日本語訳を併記

#### **3. データ・具体例による裏付け**
- 数値データで主張を補強
- 具体的な企業事例や調査結果を引用
- 「menlo venturesによると」「Anthropicの定義では」等

#### **4. 自社/自分への適用**
- 一般論を自分の文脈に落とし込む
- 「我々はこう考える」「私の経験では」
- 実践的なアプローチを示す

#### **5. Why? の問いかけ**
- 「Why 〜なのか?」という見出しで強みを説明
- 反論を予測し、先回りして対処
- 「〜という点で強みがある」

#### **6. 未来への展望・示唆**
- 今後の方向性、予測を示す
- 「〜の時代が来る」「〜が重要になる」
- 読者に行動を促す示唆

#### **7. クロージング（CTA）**
- 採用募集、カジュアル面談への誘導
- 関連記事・podcastへのリンク
- 「絶賛採用中」「ぜひ話しましょう」

### 1.3 見出し構成

**特徴:**
- H2レベル: 大きなテーマ（「AI Agentとは」「SaaS is Dead」）
- H3レベル: サブトピック（「Why バクラク?」「バクラクのチャレンジ」）
- 疑問形の見出しを効果的に使用（「Why 〜?」）

**見出しのパターン:**

1. **概念説明系**
   - 「[概念]とは」
   - 「[概念]の定義」

2. **問いかけ系**
   - 「Why [テーマ]?」
   - 「[テーマ]は本当か?」

3. **主張系**
   - 「[主張]: [補足説明]」
   - 「[テーマ]のチャレンジ」

4. **時系列系**
   - 「2025年の展望」
   - 「現在から未来へ」

## 2. トーン・文体

### 2.1 全体的なトーン

**核となるトーン:**
> 「知的な実務家が、親しみやすく、データに基づいて未来を語る」

**3つの要素:**
1. **知的**: 論理的、データドリブン、専門性
2. **親しみやすい**: カジュアルな表現、「どうも」「めちゃくちゃ」
3. **実務家視点**: 現場の経験、実践的な洞察、具体例

### 2.2 文末表現

**断定調（多用）:**
- 「〜です。」「〜だと思います。」
- 「〜になります。」「〜でしょう。」
- 自信を持った主張に使用

**思考の提示（適度に）:**
- 「〜という感覚。」「〜だと思います。」
- 「〜かなと思います。」「〜と考えています。」
- 個人的見解を示すときに使用

**疑問形（効果的に）:**
- 「〜でしょうか。」「〜なのか?」
- 読者に考えさせる、反論を予測する際に使用

**文末表現の比率（目安）:**
- 断定調: 70%
- 思考の提示: 20%
- 疑問形: 10%

### 2.3 カジュアル表現のルール

**使ってよいカジュアル表現:**
- 「めちゃくちゃ」（頻出）
- 「というか」
- 「よくよく考えると」
- 「〜だったりします」

**使わないカジュアル表現:**
- 「ヤバい」
- 「マジで」
- 絵文字（基本的に使用しない）

**カジュアルと知的のバランス:**
- オープニングはカジュアル寄り
- 本論はデータと論理で知的に
- 締めくくりは再びカジュアルに

### 2.4 視点の切り替え

**一人称の使い分け:**
- 「私」: 個人的見解、経験を語る
- 「我々」「当社」: 組織としての取り組み
- 「山内」: 自己紹介、署名

**読者への語りかけ:**
- 「〜と思いませんか?」
- 「〜という意味で重要です。」
- 「よくよく考えるべきと私は思います。」

## 3. 言語・語彙

### 3.1 専門用語の扱い

**基本ルール:**
1. **英語のまま使用**: AI Agent, Workflow, Agentic, Orchestrator, Context
2. **初出時に定義**: 括弧書きまたは別段落で説明
3. **日本語訳を併記**: 英語 → 日本語訳の順

**典型的なパターン:**

```markdown
AI Agentの定義は以下のものになります。

> [英語の引用文]
>
> (日本語訳)
> [日本語訳文]
```

**頻出する専門用語:**
- AI Agent, Agentic Workflow, Vertical AI
- Orchestrator, Workers, Context
- LLM (Large Language Model), Multimodal
- Computer Use, Voice Agent
- SaaS, BPO, ACV (Annual Contract Value)
- Hype, Deploy, Spend

### 3.2 比喩・たとえ話

**特徴:**
- 複雑な概念を身近な例でたとえる
- 「〜のようなイメージ」「〜みたいな感じ」
- 視覚的なイメージを喚起

**実例:**
- 「より柔軟なRPAみたいなイメージ」
- 「SaaSに目(AI-OCR)がついた」
- 「SaaSに頭脳がつくし、口と耳がつくし、手足も生える」
- 「大きなAIがOrchestratorとして、小さなAIをcallする」

### 3.3 特徴的なフレーズ

**頻出する接続表現:**
- 「〜という意味で」（非常に頻出）
- 「〜という事実・意味を、よくよく考えるべきと思います」
- 「〜ということだと思います」
- 「〜という点です」

**強調表現:**
- 「めちゃくちゃ〜」
- 「非常に重要な〜」
- 「衝撃的でした」
- 「まず間違いなく」

**時間・タイミングの表現:**
- 「今この瞬間だからこそ」
- 「いよいよ〜」
- 「すでに〜」
- 「2025年は〜の年になる」

## 4. データ・引用

### 4.1 数値の使い方

**特徴:**
- 具体的な数値で主張を裏付ける
- パーセンテージ、倍率、金額を明示
- 出典を必ず併記

**実例:**
- 「営業利益率55%超、平均年収日本一」
- 「SaaS $1に対して、コンサル・BPO(設定代行)に $7払われている」
- 「80%くらいまでの品質のものは速く作れるが、90%, 95%と品質を上げるにはものすごい努力が必要」

**数値を使うべき場面:**
- 市場規模、成長率
- 業務効率化の効果（時間削減、コスト削減）
- 技術の精度、性能

### 4.2 権威ある出典の引用

**頻出する引用元:**
- Anthropic（AI技術の定義）
- menlo ventures（市場データ、調査結果）
- 有名企業（Salesforce, UiPath, キーエンス）
- 学術的概念（ダイナミックスキル理論）

**引用の形式:**

```markdown
[出典]によると、[データ・主張]だそうです。

> [引用文]
```

**引用の効果:**
- 主張に権威を与える
- 客観性を担保する
- 読者の信頼を得る

## 5. 段落構成

### 5.1 段落の長さ

**基本ルール:**
- 1段落 = 1〜4文
- 短い段落と長い段落を交互に配置
- 視覚的に読みやすく

**短い段落（1〜2文）:**
- 強調したいポイント
- 段落の転換
- 問いかけ

**長い段落（3〜4文）:**
- 詳細な説明
- データの提示
- 複数の要素を列挙

### 5.2 段落間の接続

**接続詞・接続表現:**
- 「そして」「さらに」（追加）
- 「一方」「ただし」（対比）
- 「つまり」「要するに」（言い換え）
- 「なので」「その結果」（因果）

**段落の流れ:**
1. 問題提起
2. 背景・コンテキスト
3. データ・事例
4. 解釈・示唆
5. 次のトピックへ

## 6. 論理的説得

### 6.1 主張の構造

**典型的なパターン:**

```
1. 主張: 〜である。
2. 根拠: なぜなら〜だからです。
3. データ: 実際に〜というデータがあります。
4. 具体例: 例えば〜です。
5. 示唆: したがって〜ということです。
```

### 6.2 反論の予測と対処

**特徴:**
- 読者が持ちうる疑問を先回りして提示
- 「〜という反論があるかもしれません」
- その反論に対する回答を用意

**実例:**

```markdown
ここらへんの話は、実際にSaaS企業を経営している経営者からするとめちゃくちゃ反論可能だし、反論したくなるような話とも思います。SaaSはディストリビューションゲームであり、SaaSが死ぬは極論であるという反論です。

私の考えで行くと、[自分の立場]という結論です。
```

**反論対処のステップ:**
1. 反論を認識していることを示す
2. 反論の妥当性を部分的に認める
3. しかし、より本質的な視点を提示
4. 自分の結論を述べる

### 6.3 具体例による補強

**使い方:**
- 抽象的な概念を具体例で説明
- 「例えば〜」「具体的には〜」
- 読者がイメージしやすい事例

**具体例の種類:**
1. 有名企業の事例（Salesforce, キーエンス）
2. 数値データ（市場規模、効率化の効果）
3. 技術的な実装例（AI-OCR, Orchestrator-workers）
4. 日常業務の例（請求書処理、データ入力）

## 7. テーマ・トピック

### 7.1 よく扱うテーマ

**主要テーマ:**
1. **組織論**: 組織の強度、自責思考、執着心、ルール、仕組み
2. **技術トレンド**: AI Agent, 生成AI, SaaS, Vertical AI
3. **戦略論**: ポジショニング、意思決定、撤退判断、淘汰
4. **価値観**: 最適レベルと機能レベル、やりきる組織、AI-UX

### 7.2 トピックの深掘り方

**パターン:**
1. **定義から入る**: 「〜とは何か?」
2. **現状認識**: 「今、〜が起きている」
3. **問題提起**: 「しかし、〜という課題がある」
4. **解決策の提示**: 「我々は〜というアプローチをとる」
5. **未来展望**: 「今後、〜になるだろう」

## 9. 記事生成時のチェックリスト

- [ ] 親しみやすいオープニングで始まっているか

- [ ] 断定調が中心か（70%目安）

## 10. 適用ポイント

### 10.1 やるべきこと

1. **オープニングの工夫**
   - 「どうも、[ビジョン]したい山内です。」で始める
   - 時事的な背景、トレンド認識から入る

2. **データで裏付ける**
   - 主張には必ず数値・事例を添える
   - 権威ある出典を引用する

3. **断定調で書く**
   - 「〜です。」「〜だと思います。」
   - 自信を持って主張する

4. **反論を予測する**
   - 「〜という反論があるかもしれません」
   - 先回りして対処する

5. **具体例を豊富に**
   - 抽象論だけでなく、具体的な事例を
   - 読者がイメージしやすい例を選ぶ

6. **カジュアルと知的のバランス**
   - 「めちゃくちゃ」等のカジュアル表現を適度に
   - データと論理で知的に

### 10.2 避けるべきこと

1. **過度に堅苦しい文体**
   - 「〜である。」「〜であろう。」の連続
   - 論文調、お役所調

2. **過度にカジュアルな表現**
   - 「ヤバい」「マジで」
   - 絵文字の多用

3. **根拠のない主張**
   - データ・事例なしの断定
   - 「〜だと思う」だけの主張

4. **長すぎる段落**
   - 5文以上の段落
   - 視覚的に読みにくい

5. **専門用語の乱用**
   - 定義なしに専門用語を連発
   - 読者がついてこれない

## 12. 記事タイプ別の適用

### 12.1 解説・教育記事

**重点:**
- 定義の明確化
- 具体例の豊富さ
- 段階的な説明

**構成:**
1. 「[概念]とは」で定義
2. 背景・コンテキスト
3. 具体例・事例
4. 実践的な適用方法
5. まとめ・示唆

### 12.2 意見・主張記事

**重点:**
- 問題提起の鮮明さ
- データによる裏付け
- 反論への対処

**構成:**
1. 問題提起（「〜が起きている」）
2. 現状分析（データ・事例）
3. 自分の見解（「私の考えでは」）
4. 反論への対処
5. 結論・未来展望

### 12.3 ハウツー・実践記事
